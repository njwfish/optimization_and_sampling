# Figure Build Pattern

`Lecture Tex/figures/main_notes/` is the canonical home for assets used by the
main lecture notes.

`Lecture Tex/figures/handouts/` is the canonical home for assets used only by
standalone handouts.

There are two figure classes:

1. Native TikZ/PGFPlots figures.
   These are checked in as `.tex` sources under `main_notes/` and are built as
   part of the normal LaTeX compile.

2. Python-generated assets.
   These are regenerated by `Lecture Tex/figures/main_notes/build_main_note_figures.py`. At the moment
   this includes:
   - the diffusion PDFs;
   - the EM fixed-point TikZ source;
   - the ULA data tables used by the sampling TikZ figure.

The standard workflow for main-note assets is:

```bash
make figures-env
make figures
make notes
```

The `figures-env` target creates a local `.venv-figures` environment and installs
the dependencies listed in `scripts/requirements-figures.txt`. The `figures`
target automatically uses that environment when it exists, and otherwise falls
back to the system `python3`.

Handout-only assets should live under `handouts/<handout_slug>/` and should be
included explicitly as `\input{figures/handouts/<handout_slug>/fig_<name>}` or
`\includegraphics{figures/handouts/<handout_slug>/fig_<name>}`.

Current generator entrypoints are:

- `python3 "Lecture Tex/figures/handouts/convexity_optimization/make_convexity_optimization_figures.py"`
- `python3 "Lecture Tex/figures/main_notes/augmented_state_optimization/make_em_figures.py"`
- `python3 "Lecture Tex/figures/main_notes/sampling/build_parallel_mcmc_ula_figures.py"`
- `python3 "Lecture Tex/figures/handouts/discrete_diffusion/make_discrete_diffusion_figures.py"`
- `python3 "Lecture Tex/figures/handouts/elbo_geometry/make_elbo_geometry_figures.py"`
- `python3 "Lecture Tex/figures/handouts/langevin_entropy_transport/make_langevin_entropy_transport_figures.py"`

When a generator emits both a shared main-note asset and handout-only outputs,
it should live next to the canonical shared asset rather than in a generic
scripts directory. That is why the EM generator lives under
`main_notes/augmented_state_optimization/`.

If a handout reuses a figure or data table that is already part of the main-note
build, keep the canonical asset under `main_notes/<unit>/` and reference it from
the handout rather than duplicating it.

Do not add new figure assets directly at the top level of `Lecture Tex/figures/`.
Any remaining top-level files are migration leftovers and should be sorted into
`main_notes/`, `handouts/`, or a clearly named archival location before they
grow further.
